## ADDED Requirements

### Requirement: The operator UI SHALL provide a read-only daily decision page rendering persisted artifacts only

The operator UI SHALL add a "今日推荐" page under the "运行" navigation group.
The page SHALL only render persisted artifacts — the dated
`daily_recommendation_*.json` files and the production model's meta sidecars —
and SHALL never re-run inference, trigger training/GPU work, or edit any run
configuration. Apart from decision-journal appends (below) the page is
read-only. The page SHALL follow the established module pattern: pure helpers
in `pages/_daily_decision_helpers.py`, journal I/O in
`web/operator_ui/decision_journal.py` (mirroring `job_io.py`), thin Streamlit
rendering in `pages/daily_decision.py`, artifact access via `artifact_reader` +
`_path_guard`.

#### Scenario: page renders from disk only
- **WHEN** the operator opens 今日推荐
- **THEN** the candidate table is populated from the selected
  `daily_recommendation_{date}.json` (default: latest available date) and no
  inference, training or job-launch code path is invoked

#### Scenario: no artifacts yet
- **WHEN** `output/daily_recommend/` contains no recommendation JSON
- **THEN** the page shows an empty state pointing at
  `scripts/daily_recommend.py` — it does not offer to run it

### Requirement: The model-meta banner SHALL warn on any missing field and never substitute defaults

The page SHALL render a persistent banner sourced from the production model's
promotion meta (`<stem>.meta.json`, resolved from `QUANT_MODEL_PATH`) showing:
`fit_end_for_inference`, the training window (`train_window`), the promotion
date (`promoted_at`) and the model identity (`model_path` + `model_type`). If
ANY of these fields is missing — or the meta file itself is absent — the banner
SHALL render a prominent WARN state naming the missing field(s); it SHALL NOT
render a default, placeholder or inferred value in place of a missing field.
The banner SHALL additionally cross-check the selected artifact against the
current model: `meta.model_pkl_sha256` (artifact) vs `pkl_sha256` (trainer
sidecar of the current model) — a mismatch renders a WARN that the artifact was
generated by a different model; a v1 artifact (no `meta` block) renders a WARN
that generation context is unavailable.

#### Scenario: missing meta field warns loudly
- **WHEN** the promotion meta lacks `fit_end_for_inference`
- **THEN** the banner shows a WARN naming `fit_end_for_inference` and shows no
  substitute value for it

#### Scenario: stale artifact is flagged
- **WHEN** the selected artifact's `meta.model_pkl_sha256` differs from the
  current model sidecar's `pkl_sha256`
- **THEN** the page shows a WARN stating the artifact was generated by a
  different model than the one currently promoted

### Requirement: The candidate table SHALL pass through existing flags only

The candidate table SHALL render, per pick: rank, stock code, stock name,
predicted score, `tradable_flag`, `unavailable_reason` (which already carries
`st` / suspension / one-price-lock reasons from the generation side), plus a
display-only cost reference column comparing the predicted score against a
30 bps round-trip cost constant. The UI SHALL NOT compute any new
ST/suspension/PIT flag on its own; absent source fields render as absent.

#### Scenario: flags are pass-through
- **WHEN** a pick carries `tradable_flag=false` with `unavailable_reason`
- **THEN** the table shows exactly those values; no UI-side recomputation occurs

### Requirement: The decision journal SHALL be an append-only JSONL owned by the web layer

Decision entries SHALL be appended to `decision_journal.jsonl` under the
directory resolved from `QUANT_DECISION_JOURNAL_DIR` (default
`D:/stock/operator_journal`, outside the repository's disposable `output/`
tree; the variable SHALL be registered in `docs/operations-env-vars.md`). Each
line is a single JSON object:
`{"journal_version": 1, "trade_date", "code", "action": "adopt|reject|watch",
"reason", "rank", "score", "model_id", "decided_at"}` where `trade_date` SHALL
be copied from the selected artifact's `as_of_date` (never the local clock) and
`decided_at` SHALL be an Asia/Shanghai ISO8601 timestamp with explicit offset.
The file SHALL be UTF-8 without BOM with `\n` line endings written in binary
mode. Entries are never modified or deleted; a correction is a NEW appended
entry, and the read function SHALL implement supersede semantics: for the same
`(trade_date, code)` the entry with the latest `decided_at` wins.

#### Scenario: correction supersedes, history survives
- **WHEN** the operator records `adopt` then later `reject` for the same
  `(trade_date, code)`
- **THEN** the file contains BOTH lines and the read function returns `reject`
  as the effective decision

#### Scenario: line endings are clean bytes
- **WHEN** any entry is appended on Windows
- **THEN** the file contains no `\r` byte and every line ends with exactly
  `}\n` (byte-level assertion)

### Requirement: Journal writes SHALL be idempotent and partial-write tolerant

Submission SHALL go through an explicit button; the form SHALL mint a uuid4
nonce at render time (held in session state), and the append function SHALL
refuse to append when an entry with the same `(trade_date, code, nonce)`
already exists in the file — so a Streamlit rerun replaying the same submission
cannot produce a duplicate line. Each append SHALL build the complete line
(including the trailing `\n`) before a single write+flush on a binary append
handle, so an interrupted process never leaves a torn line mid-file. The reader
SHALL skip malformed lines without crashing, count them, and the page SHALL
surface the malformed-line count as a WARN.

#### Scenario: rerun does not duplicate
- **WHEN** the same `(trade_date, code, nonce)` append is invoked twice
- **THEN** the journal contains exactly one matching line

#### Scenario: a torn line does not poison the journal
- **WHEN** the journal contains a truncated (non-JSON) line among valid lines
- **THEN** reading returns all valid entries plus a malformed-line count, and
  the page renders a WARN with that count

### Requirement: The journal SHALL never feed official metrics or src/ code

The decision journal SHALL remain operator-owned web-layer state: no module
under `src/` may reference it (by symbol, filename or env var), and it SHALL
never be an input to official metrics, backtests, training or promotion
decisions. A source-scan test in `tests/logic` SHALL enforce zero `src/`
references, and `web/README.md` SHALL state this boundary.

#### Scenario: boundary is machine-enforced
- **WHEN** any `src/` module gains a reference to `decision_journal` (symbol,
  filename or env var)
- **THEN** the source-scan test fails
