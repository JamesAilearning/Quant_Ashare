## ADDED Requirements

### Requirement: Operator UI SHALL list reusable UI-managed providers

The operator UI SHALL list reusable qlib providers previously generated by
UI-managed Tushare provider jobs. The list SHALL be built only from
`output/operator_ui/results/*/qlib_provider` directories and SHALL display
read-only provider metadata when available. Listing providers SHALL NOT
initialize qlib, call Tushare, or compute runtime metrics.

#### Scenario: reusable Tushare provider exists

- **WHEN** `output/operator_ui/results/tushare_provider_20260517_000000_abcd1234/qlib_provider` exists
- **AND** adjacent provider metadata artifacts exist
- **THEN** Config & Run displays that provider as a selectable data source
- **AND** the option includes coverage and health metadata

---

### Requirement: Operator UI SHALL populate provider_uri from selected provider

When the operator selects a reusable provider, the operator UI SHALL populate
the existing `provider_uri` input with that provider's `qlib_provider` path.
The UI SHALL keep manual provider URI entry available and SHALL continue to run
existing provider validation against the final `provider_uri` value.

#### Scenario: operator selects an existing provider

- **WHEN** an operator selects a reusable provider from the provider picker
- **THEN** `provider_uri` is set to that provider's `qlib_provider` path
- **AND** Provider Preview and Run validation use that same path

#### Scenario: operator chooses manual entry

- **WHEN** an operator chooses manual provider entry
- **THEN** the `provider_uri` input remains editable
- **AND** no hidden fallback provider path is selected

---

### Requirement: Operator UI SHALL delete saved provider data explicitly

The operator UI SHALL provide an explicit delete action for UI-managed saved
provider data. The delete action SHALL remove only the selected
`output/operator_ui/results/<job_id>` result directory when that directory
contains a `qlib_provider` subdirectory. The delete action SHALL reject path
traversal and SHALL NOT delete arbitrary machine-local provider paths.

#### Scenario: operator deletes a saved provider

- **WHEN** an operator selects a reusable provider
- **AND** chooses the delete action for that provider
- **THEN** the UI deletes that provider's UI-managed result directory
- **AND** clears the selected provider URI state

#### Scenario: delete target is not a saved provider

- **WHEN** a delete request targets a result directory without `qlib_provider`
- **THEN** the UI refuses the delete
- **AND** the target directory remains present

---

### Requirement: Operator UI SHALL delete non-running recent jobs explicitly

The operator UI SHALL provide an explicit delete action for recent UI jobs. The
delete action SHALL remove only the selected `output/operator_ui/jobs/<job_id>`
job directory and SHALL reject path traversal. The UI SHALL NOT delete a job
whose status is `running`; operators must stop running jobs before deleting the
job record and logs.

#### Scenario: operator deletes a finished job

- **WHEN** a UI job status is not `running`
- **AND** the operator chooses the delete action for that job
- **THEN** the UI deletes that job's job directory

#### Scenario: operator tries to delete a running job

- **WHEN** a UI job status is `running`
- **THEN** the delete action is disabled or refused
- **AND** the running job directory remains present
