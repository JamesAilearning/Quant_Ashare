## 0. 这是什么 / 不是什么

REGEN-A 用**冻结分数纯重放**量化"修正回测语义"对 walk-forward 基线的影响。**零重训、零 bundle 触碰**。

- **方法**：加载 0.301 轮（C1）的 22 折冻结预测 `.pkl`（post-ensemble 信号 Series，与回测真正消费的完全一致）→ 喂进当前 main 的 `BacktestRunner`（canonical 设置）→ 按 `nanmean(per-fold with-cost IR)` 聚合。模型分数一字未动，只换回测语义。
- **不是重跑 walk-forward**：没有训练任何模型、没有重建任何特征、没有碰 bundle。
- **bundle 锁**（跑前再确认）：`D:/qlib_data/my_cn_data_pit` 仍是 **5-28 vintage**（1942 个交易日、末日 2025-12-31、无 `benchmark.txt`、无 `features/sh000300tr/`）。与冻结分数同源，重放有效。

## 1. 修正了什么（两个度量 bug，各自独立核实）

| 修复 | 旧行为（C1=0.301 轮实测） | 新行为（当前 main） | 实证 |
|---|---|---|---|
| **PR-C 执行时点** | `lag=1` 实测 **T+2 成交**（信号 04-01 → 成交 04-03，跳过 04-02） | `lag=1` = **T+1**（信号 04-01 → 成交 04-02） | 控制实验强制单票置顶看首次成交日 |
| **PR-D 涨跌停** | `0.095` 当 float 喂入 → PIT bundle 无 `$change` → **什么都不挡**，涨停可买 | 收盘价表达式 `Not($close/Ref($close,1)-1 ≤ mag)` → **真挡涨停**（每折 ~36-297 个真涨停日） | 逐折统计真封单数 |

**关键定性（③ 口径，必须写死）**：修正基线**上移 = 度量修对，不是策略变好。**
- 旧 T+2 **低估**了策略——它用了陈旧一天的信号；实盘 `daily_recommend` 一直是 T+1 进场。所以修正后的数字才是和**实盘口径一致**的度量，旧 0.301/0.367 是 T+2-buggy 的低估值 + 涨停虚拟成交的虚高值的混合。
- **不预示实盘会更好**：这是把回测度量校准到实盘本就在做的 T+1，不是策略 alpha 提升。
- **且头条上移本身落在噪声内、离群驱动**（§3 关键）：22 折 11↑11↓、中位 Δ≈0、均值移动≈1 个标准误、剔 3 个离群折即反转。所以**连"IR 升了"都不该当成稳健改善**——稳健的表述只有一句：度量口径现在对了（T+1 + 真挡涨停）。
- **基准仍是价格指数 SH000300（临时）**：TR 全收益延到 REGEN-2。届时超额年化预计**下修 ~2-2.5pp**（股息拖累）。本版所有"超额"数字是价格指数口径，**不可与 REGEN-2 直比**。

**判定：离群驱动，不是普涨。**
- **过半折持平或下行：11 折↑ / 11 折↓，中位 Δ = −0.05**（中位折几乎不动）。
- 头条均值上移 +0.178 全靠少数大正离群折扛起：**fold 12(+2.59)、11(+1.54)、10(+1.15)、5(+1.03)**；**剔掉前 3 个正离群折（12/11/10），均值移动反转为 −0.07**。
- **均值移动落在噪声内**：mean Δ / SE = **0.99**（≈1 个标准误，统计上与 0 不可区分）；跨折 IR 离散 std≈1.90、均值 SE≈0.41，0.18 的移动远在此带内。
- 因此**不要把"IR 从 0.30 升到 0.48"读成策略变好或更稳**——这是 ① 度量修对（个别动量折因 T+1 收回 alpha 而跳高）叠加 ② 涨停折被正确压低，净在噪声内的重排。

   - ④ 注：2018 去杠杆、2020-02 COVID 暴跌均落在**训练窗**、不在任何测试折（最早测试折 = 2020-Q2）；已扫遍所有 22 个测试窗，含 2020-Q2/2024-Q3 等停牌/涨停高发期。

## 5b. 研究方法债（记一笔，影响后续比较）

**WF 头条 mean-fold-IR 的标准误 ≈ 0.41**（22 折、跨折 IR std≈1.9）。含义：
- 任何 **< ~0.4 量级的 mean-IR 差异在 WF 层不可检测**（落在一个标准误内）。本次"修正"上移 0.18 即是例证——统计上与 0 不可区分。
- **影响阶段 6（标签跨度实验）**：span=5/7/10 的变体若只看 WF mean-IR 差，几乎肯定测不出来——需要配对/逐折检验或更稳的统计量（如逐折 IC 的配对 t、或 block bootstrap），不能只比头条均值。
- **影响 C2 类比较**（任何"改了某项→看 WF 基线动没动"）：同理，单点 mean-IR 比较的功效极低。后续凡是要"证明某改动改善了基线"，必须报效应量 + 不确定带，而非单点。
- 根因：日频 csi300 walk-forward 每折 IR 本身高方差（市场 regime 主导），22 折不足以把均值标准误压到小效应可检。这是研究设计层面的债，不是某个 PR 的 bug。
